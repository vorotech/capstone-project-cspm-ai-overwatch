## Головне дослідницьке питання

**Яким чином відрізняється здатність різних архітектур великих мовних моделей (LLM) ефективно виконувати роль контекстно-залежного хмарного аудитора, забезпечуючи високу стабільність аналізу результатів CSPM-інструментів з урахуванням широкого архітектурного контексту для відсіювання хибних спрацьовувань та обґрунтованого коригування рівня ризику?**

## Огляд проєкту
**Мета дослідження**: Порівняльний аналіз LLM у задачах контекстного відсіювання False Positives та валідації компенсуючих заходів безпеки, які не здатні зафіксувати стандартні CSPM-інструменти.

**Суть проєкту**: Дослідження здатності LLM аналізувати не просто окремі алерти CSPM, а також архітектурний контекст (в майбутньому із врахуванням коду конфігурації, але в цьому дослідженні код не надається).

* **Поетапний пайплайн**: Гнучкий набір скриптів для розгортання інфраструктури (Terraform), збору результатів (Prowler/AWS Security Hub) та багаторазового прогону LLM для накопичення статистики.

* **Анонімізація даних (Data Sanitization)**: Перед відправкою контексту до зовнішніх API LLM всі чутливі ідентифікатори (зокрема AWS Account IDs) замінюються на маски (`<REDACTED>`), забезпечуючи безпеку в межах Trust Boundary.

* **Аналітика та Dashboard**: Пайплайн розраховує ключові метрики (Adjustment Rate, Decision Consistency) та відображає зібрану статистику у вигляді інтерактивной презентації на [слайді 15](https://vorotech.github.io/capstone-project-cspm-ai-overwatch/?slide=15) (Vite + Recharts).

## Бюджет та Витрати (Budget & Costs)

> [!WARNING]
> Запуск цього пайплайну вимагає фінансових витрат. Детальний прорахунок буде додано після завершення замірів, але варто враховувати наступне:
> - **AWS**: Створення ресурсів через Terraform є платним. Дещо може покриватися Free Tier, але загальна вартість залежатиме від кількості та часу існування конфігурацій до виконання `terraform destroy`.
> - **OpenRouter (LLM)**: Для аналізу результатів використовуються платні моделі. Бажано мати баланс більше $10 на акаунті OpenRouter, оскільки це знімає жорсткі ліміти швидкості (rate limits) на запити до платних моделей.
